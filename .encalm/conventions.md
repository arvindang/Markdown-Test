# Style Guide

## Document Structure

### Heading Hierarchy
- Use standard Markdown hierarchy: `#` > `##` > `###`
- Document titles use H1 (`#`)
- Main sections use H2 (`##`)
- Subsections use H3 (`###`)

### Document Metadata
- Include document filename in parentheses after title
- Format: `## Title (filename.md)`

## Content Formatting

### Lists
- Use asterisk (`*`) for unordered lists
- Single space after bullet point
- No blank lines between simple list items

### Emphasis
- Use single asterisks for italic emphasis: `*italic*`
- Bold formatting appears to use asterisks as well: `*bold*`

### Code and Technical Elements
- Inline code uses backticks: `` `code` ``

## Comments and Annotations

### HTML Comments
- Use HTML comment syntax for editorial notes: `[//]: # (comment text)`
- Place comments on separate lines
- Use sentence case, end with period for complete thoughts
- Common uses:
  - Content update instructions
  - Editorial notes
  - Inline documentation

## Writing Style

### Tone
- Conversational and direct
- Use first person ("we") for collaborative tone
- Ask rhetorical questions to engage reader

### Sentence Structure
- Mix of short and longer sentences
- Incomplete sentences acceptable for emphasis or style
- Stream-of-consciousness passages allowed

### Content Approach
- Test content and placeholders are acceptable
- Works-in-progress formatting preserved
- Multiple perspectives encouraged in analysis

## Spacing and Layout

### Paragraph Breaks
- Single blank line between paragraphs
- Multiple blank lines used for major section breaks
- Horizontal rules (`---`) for document separators

### Indentation
- No special indentation requirements observed
- Standard Markdown formatting sufficient

## Special Conventions

### Document Separation
- Use `---` horizontal rule to separate multiple documents
- Include source filename for each document section

This style guide reflects a flexible, work-in-progress documentation approach that prioritizes readability and collaborative editing over rigid formatting rules.
